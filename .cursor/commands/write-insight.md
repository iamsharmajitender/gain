# Write insight (create new article)

Read and follow **`.cursor/skills/write-insight/SKILL.md`** completely.

**Phase 1 only** unless the user already replied **`approve`**.

1. From the user's brief (or attached context), classify article type using `insights-article-structure` rule.
2. Propose slug, folder path, frontmatter, tags, section outline, **insights-only** outbound links (scan `docs/insights/` only), and hero PNG spec.
3. Return the **Proposed insight** report. **Do not create files yet.**

End by asking the user to reply **`approve`** to create the scaffold, request edits, or **`abort`** to cancel with no changes.
