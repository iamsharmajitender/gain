## Operating model planes

Component-level architecture for each plane: who builds what and how they connect.

- **How to Model AI Control Plane**: gateway, policy, routing, tools, observability, evaluation
- **How to Model Context Engine**: retrieval orchestration; data plane → control plane boundary
- **How to Model Data / Knowledge Plane**: pipelines, embeddings, catalogs, governance tags
- **How to Model AI Runtime Plane**: GPU clusters, model serving, vector stores, networking

## Capability patterns

G.A.I.N patterns operationalized on the control plane.

- **How to Model LLM Gateway**: G.A.I.N LLM: governed model access, guardrails, caching
- **How to Model RAG Architecture**: G.A.I.N RAG: indexing, retrieval, reranking, generation
- **How to Model Agent Flow**: G.A.I.N Agents: planning, tools, policy-governed execution

## Reference models

- **How to Model G.A.I.N Architecture**: layered reference model from ingress through intelligence, data, and cloud
- **How to Model Memory**: short-term, long-term, and episodic memory for agents and conversational systems
- **How to Model Decision-Making**: reasoning chains, policy gates, and human-in-the-loop workflows
