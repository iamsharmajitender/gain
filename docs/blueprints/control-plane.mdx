## Plane placement

| Attribute | Value |
| --- | --- |
| **G.A.I.N pillar** | Intelligent (primary), Adaptive (feedback hooks) |
| **Owner** | AI Platform Team |
| **Consumes** | Runtime plane (inference, vector stores), Data plane (indexed knowledge via Context Engine) |
| **Consumed by** | Application plane (copilots, workflows, domain agents) |

## Core components

<Details className="gain-accordion" summary="AI Gateway: standard entrypoint">
  Single front door for all AI traffic: human and system callers. Responsibilities:

  - Authentication and authorization (integrate with enterprise IdP)
  - Rate limiting and quota enforcement per tenant / use case
  - Request/response schema validation and protocol translation (REST, gRPC, SSE)
  - Correlation and trace ID propagation for observability
  - Versioned public API surface: product teams never call model vendors directly

  **Does not own:** business workflows, domain prompts, source-system integration.
</Details>

<Details className="gain-accordion" summary="Model abstraction layer: provider swapability">
  Abstraction over OpenAI, Anthropic, Azure, Bedrock, and on-prem inference. Responsibilities:

  - Unified invocation contract (prompt + tools + schema + metadata)
  - Provider failover and circuit breaking
  - Token accounting and cost attribution tags
  - Model capability matrix (context window, tool support, streaming)

  **Interface to runtime plane:** calls inference endpoints; never embeds vendor SDKs in application code.
</Details>

<Details className="gain-accordion" summary="Intent classifier: routing intelligence">
  Classifies incoming requests before expensive inference. Responsibilities:

  - Route to capability pattern (chat, RAG, agent, structured extraction)
  - Detect out-of-scope or high-risk intents for policy escalation
  - Select retrieval profile or agent profile for downstream orchestration

  **TBD:** fine-tuned classifier vs. LLM-based routing vs. rules engine hybrid.
</Details>

<Details className="gain-accordion" summary="Policy engine: governance">
  Deterministic enforcement layer: authorization is not prediction. Responsibilities:

  - Data classification checks (PII, sensitive, regulated)
  - Model allowlists / blocklists per use case
  - Tool permission scopes and four-eyes approval hooks
  - Prompt injection and exfiltration guardrails
  - Audit event emission for every policy decision

  **Shared with Security:** policy semantics; Platform owns runtime enforcement.
</Details>

<Details className="gain-accordion" summary="Routing engine: cost, latency, risk">
  Selects model and inference path after policy clears. Decision inputs:

  - Latency SLA, cost budget, quality tier
  - Data residency and air-gap constraints
  - Fallback chain when primary model degrades

  **Does not own:** embeddings pipelines, catalog semantics (Data plane).
</Details>

<Details className="gain-accordion" summary="Prompt templates: standardized invocation">
  Versioned prompt contracts: treated as platform APIs. Responsibilities:

  - Platform-owned system prompts and output schemas
  - Variable injection and safe templating
  - A/B and regression hooks tied to evaluation framework

  **Boundary:** business-specific prompt content stays in Application plane; platform provides shells and guardrails.
</Details>

<Details className="gain-accordion" summary="Tool registry: reusable enterprise tools">
  Catalog of approved tools agents and workflows may invoke. Each tool entry includes:

  - Typed input/output schema
  - Permission scope and owning domain team
  - Audit trail and rate limits
  - Sandbox vs. production endpoints

  See [How to Model Agent Flow](/blueprints/agent-flow-model) for execution semantics.
</Details>

<Details className="gain-accordion" summary="Observability: tracing, token cost, quality">
  Cross-cutting telemetry for every control-plane hop. Capture five signals in the request path and
  route to purpose-built tiers:

  - **Structured logs** → log store (30 d): operational debugging
  - **Metrics** → TSDB (13 mo): SLOs, token cost, latency percentiles
  - **Sampled traces** → trace store (30 d): gateway → context → model → tools
  - **Raw prompt/response** → restricted store (encrypted, 90 d): quality and drift analysis
  - **Audit records** → audit log (immutable, 7 y): compliance replay

  Redaction and sampling happen in the collector. See [G.A.I.N Observability](/frameworks/gain-observability).

  **Spans all planes**: collectors live in platform; storage may sit on runtime infra.
</Details>

<Details className="gain-accordion" summary="Evaluation framework: benchmark and regression">
  Continuous quality gate for prompts, models, and retrieval profiles. Responsibilities:

  - Golden datasets per capability pattern
  - Regression runs on model or index changes
  - Grounding and citation accuracy checks for RAG
  - Agent task success rates and policy violation counts

  **Feeds Adaptive pillar:** evaluation results inform product KPIs and platform tuning.
</Details>

<Details className="gain-accordion" summary="Secrets integration: secure model credentials">
  Centralized credential brokering: no raw API keys in application repos. Responsibilities:

  - Vault integration for provider keys and service accounts
  - Short-lived tokens injected at gateway boundary
  - Rotation and break-glass procedures

  **Runtime plane** hosts vault agents; **Platform** owns consumption contract.
</Details>
